# Upbit AI Trader 작업내역

작업일: 2026-03-29

## 목적

프런트 운영 안정성 보강, 로컬/프록시 환경에서의 API·WebSocket 연결 정리, 일일 리포트 backfill 경로 분리를 한 묶음으로 정리했다.

## 한눈에 보기

- 프런트가 현재 브라우저 기준 same-origin으로 API / WebSocket 주소를 우선 사용하게 바꿨다.
- 자동매매 상태와 실시간 연결 상태의 초기값을 `null`로 분리해, 첫 렌더에서 `OFF`나 `끊김`으로 오판하지 않게 했다.
- 전역 `WSInitializer`를 `providers`로 올려 대시보드 외 페이지에서도 동일한 실시간 상태를 유지하게 했다.
- 차트 컴포넌트의 비동기 초기화 경합과 로드 실패 UI를 보강했다.
- 백테스트/성과 비교 지표에서 `null` 값 때문에 `NaN`, `null건`, 타입 오류가 나는 부분을 정리했다.
- `daily-report/history`는 읽기 전용으로 되돌리고, 별도 `POST /api/v1/portfolio/daily-report/backfill` 관리자 경로를 추가했다.
- 운영 문서, 체크리스트, 사용자 매뉴얼을 현재 코드 기준으로 다시 맞췄다.

## 반영한 수정

### 1. 프런트 기본 연결 경로 정리

- 파일: `docker-compose.yml`
- 파일: `frontend/Dockerfile`
- 파일: `frontend/src/services/api.ts`
- 파일: `frontend/src/hooks/useUpbitWS.ts`
- compose 기본 게이트웨이 포트를 `8001` 기준으로 정리했다.
- 프런트는 `NEXT_PUBLIC_API_URL` / `NEXT_PUBLIC_WS_URL`가 없을 때 브라우저 `origin/host`를 기준으로 same-origin 연결을 우선 사용한다.
- 로컬 브라우저, NAS 프록시, 다른 기기 접속 시 주소 불일치를 줄였다.

### 2. 전역 실시간 상태 초기화 정리

- 파일: `frontend/src/app/providers.tsx`
- 파일: `frontend/src/app/(dashboard)/layout.tsx`
- 파일: `frontend/src/components/layout/Sidebar.tsx`
- 파일: `frontend/src/components/layout/AutoTradeToggle.tsx`
- 파일: `frontend/src/components/layout/GlobalHeader.tsx`
- 파일: `frontend/src/store/useMarketStore.ts`
- 파일: `frontend/src/store/useTradeStore.ts`
- `WSInitializer`를 대시보드 레이아웃이 아니라 전역 providers로 이동했다.
- 연결 상태와 자동매매 상태의 초기값을 `null`로 분리했다.
- UI에는 `연결 확인 중`, `자동매매 확인 중` 상태를 표시하고, 상태를 모를 때 `OFF`로 단정하지 않도록 정리했다.

### 3. 차트 초기화와 오류 표시 보강

- 파일: `frontend/src/components/charts/TradingViewChart.tsx`
- 파일: `frontend/src/components/dashboard/EquityCurvePanel.tsx`
- 파일: `frontend/src/app/performance/market/[market]/page.tsx`
- `lightweight-charts` 동적 import 이후 첫 렌더 경합을 줄이기 위해 `isChartReady` 상태를 추가했다.
- `ResizeObserver` cleanup과 언마운트 후 state 갱신 경로를 정리했다.
- 차트 엔진 또는 데이터 로드 실패 시 화면에 오류 문구를 직접 표시하도록 바꿨다.

### 4. 성과/백테스트 숫자 null-safe 처리

- 파일: `frontend/src/app/backtest/page.tsx`
- 파일: `frontend/src/components/dashboard/PerformancePanel.tsx`
- 파일: `frontend/src/app/performance/market/[market]/page.tsx`
- 파일: `frontend/src/types/market.ts`
- 백테스트 지표가 `null`일 때도 안전하게 렌더되도록 타입과 포맷 함수를 보강했다.
- 실거래 대비 비교 카드에서 `delta` 계산이 불가능한 값은 `-`로 표시하게 정리했다.

### 5. 일일 리포트 backfill 구조 분리

- 파일: `backend/apps/gateway/api/v1/portfolio.py`
- 파일: `backend/tests/unit/test_portfolio_api.py`
- `GET /api/v1/portfolio/daily-report/history`는 다시 읽기 전용으로 돌렸다.
- 별도 관리자 경로 `POST /api/v1/portfolio/daily-report/backfill`를 추가했다.
- 이 관리자 경로는 과거 snapshot과 `risk.daily_pnl.YYYYMMDD` runtime_state를 재계산해 갱신하고, `daily_report_backfilled` 감사로그를 남긴다.
