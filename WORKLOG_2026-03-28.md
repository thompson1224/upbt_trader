# Upbit AI Trader 작업내역

작업일: 2026-03-28

## 목적

프런트 운영 안정성과 `docker compose` 기본 실행 경로를 정리하고, 운영 문서와 실제 런타임 동작을 맞췄다.

## 한눈에 보기

- `docker compose` 기본 게이트웨이 포트를 `8001`로 통일했다.
- 프런트가 현재 브라우저 호스트 기준으로 API / WebSocket 주소를 자동 추적하도록 바꿨다.
- 전역 WebSocket 초기화, 재연결, cleanup 경로를 정리해 페이지 이동/재마운트 시 누수와 끊김을 줄였다.
- 대시보드와 성과 화면의 차트 초기화 경합, 백테스트/실거래 비교 지표의 `null` 처리 문제를 보수했다.
- `연결 확인 중`, `자동매매 확인 중` 같은 초기 상태 UX를 추가하고, 자동매매 상태를 확인할 수 없을 때는 `OFF`로 단정하지 않고 `확인 중` 상태를 유지하도록 바꿨다.
- 운영 문서와 사용자 문서를 현재 기본 포트/검증 절차에 맞게 업데이트했다.
- 루트 통합 검증 스크립트 `./scripts/verify_all.sh`를 추가했고, 백엔드 pytest도 plain 실행만으로 통과하도록 정리했다.

## 반영한 수정

### 1. compose 기본 게이트웨이 포트 정리

- 파일: `docker-compose.yml`
- 파일: `frontend/Dockerfile`
- `docker compose` 기본 게이트웨이 노출 포트를 `8000` 대신 `8001`로 정리했다.
- 프런트 빌드/런타임 환경변수도 같은 기본값을 사용하도록 맞췄다.

### 2. 브라우저 호스트 기반 API / WebSocket 주소 추적

- 파일: `frontend/src/services/api.ts`
- 파일: `frontend/src/hooks/useUpbitWS.ts`
- 프런트가 `localhost` 고정 주소만 보지 않고, 현재 브라우저가 접속한 호스트/IP를 기준으로 API와 WebSocket 주소를 자동 보정하도록 변경했다.
- 로컬 `docker compose`, 직접 실행, 같은 네트워크의 다른 기기 접속 시 경로 불일치를 줄였다.

### 3. 전역 WebSocket 초기화 정리

- 파일: `frontend/src/app/(dashboard)/layout.tsx`
- 파일: `frontend/src/app/providers.tsx`
- `WSInitializer`를 대시보드 레이아웃이 아니라 전역 `providers`로 이동했다.
- 성과 상세 페이지, 설정 페이지처럼 대시보드 외 경로에서도 동일한 실시간 상태를 공유하도록 맞췄다.

### 4. 연결 상태 / 자동매매 상태 초기 UX 보강

- 파일: `frontend/src/components/layout/Sidebar.tsx`
- 파일: `frontend/src/components/layout/AutoTradeToggle.tsx`
- 파일: `frontend/src/components/layout/GlobalHeader.tsx`
- 파일: `frontend/src/store/useMarketStore.ts`
- 파일: `frontend/src/store/useTradeStore.ts`
- 연결 상태와 자동매매 상태의 초기값을 `false` 대신 `null`로 분리해, 첫 렌더에서 즉시 실패처럼 보이지 않게 했다.
- UI에는 `연결 확인 중`, `자동매매 확인 중` 상태를 표시한다.
- 자동매매 상태를 읽지 못한 경우에는 `OFF`로 오표시하지 않고 `확인 중` 상태를 유지한다.

### 5. 차트 컴포넌트 비동기 초기화 경합 수정

- 파일: `frontend/src/components/charts/TradingViewChart.tsx`
- 파일: `frontend/src/components/dashboard/EquityCurvePanel.tsx`
- 파일: `frontend/src/app/performance/market/[market]/page.tsx`
- `lightweight-charts` 동적 import 뒤에 첫 데이터 로드가 누락되거나, 첫 렌더에 빈 데이터면 차트가 영영 뜨지 않을 수 있던 문제를 수정했다.
- `ResizeObserver` cleanup 누락도 같이 정리했다.
- 차트 엔진/데이터 로드 실패 시 사용자에게 오류 메시지를 표시한다.

### 6. WebSocket 재연결 및 cleanup 안정화

- 파일: `frontend/src/hooks/useUpbitWS.ts`
- cleanup 이후에도 `onclose`가 재연결 타이머를 다시 거는 문제를 막았다.
- `signals` 채널은 원래 재연결이 없어서 한번 끊기면 새 신호가 멈췄는데, 자동 재접속을 추가했다.

### 7. 백테스트 / 성과 비교 지표 null 처리

- 파일: `frontend/src/types/market.ts`
- 파일: `frontend/src/app/backtest/page.tsx`
- 파일: `frontend/src/components/dashboard/PerformancePanel.tsx`
- 파일: `frontend/src/app/performance/market/[market]/page.tsx`
- 백테스트 지표가 `null`일 수 있는 실제 응답에 맞춰 타입을 수정했다.
- 승률, Profit Factor, 거래 수 비교에서 `NaN`, `null건`, 렌더 예외가 발생하지 않도록 포맷과 delta 계산을 보강했다.

### 8. 운영 문서 및 스모크체크 정리

- 파일: `scripts/ops_smoke_check.sh`
- 파일: `scripts/verify_all.sh`
- 파일: `backend/tests/conftest.py`
- 파일: `USER_MANUAL.md`
- 파일: `OPERATIONS_RUNBOOK.md`
- 파일: `FINAL_CHECKLIST.md`
- 스모크체크에 성과 API, 프런트 호스트 라우트, 프런트 내부 서비스 접근 확인을 추가했다.
- 루트에서 재현 가능한 검증 진입점 `./scripts/verify_all.sh`를 추가해 프런트, 백엔드, compose 검증을 한 번에 실행할 수 있게 했다.
- 통합 검증의 프런트 빌드는 Turbopack 샌드박스 제약 영향을 줄이기 위해 `next build --webpack` 경로를 사용한다.
- pytest가 기본 실행만으로도 `apps`, `libs` 패키지를 찾도록 테스트 루트 `conftest.py`에서 백엔드 루트를 `sys.path`에 추가했다.
- 운영/사용 문서에 `8001` 기본 포트, 브라우저 호스트 추적, 초기 상태 표시 의미를 반영했다.

## 최종 검증

- `frontend`: `npm run lint`
- `frontend`: `npm run build`
- `frontend`: `npx next build --webpack`
- `backend`: `./.venv/bin/pytest tests/unit`
- 루트: `sh scripts/ops_smoke_check.sh`
- 루트: `./scripts/verify_all.sh`

검증 결과:

- 프런트 lint 통과
- 프런트 build 통과
- 백엔드 unit test `95 passed`
- compose smoke 통과
- 통합 검증 스크립트 `./scripts/verify_all.sh` 끝까지 통과

참고:

- Next Turbopack 빌드는 샌드박스/환경 제약으로 불안정할 수 있어, 통합 검증에서는 `next build --webpack` 경로를 사용한다.
- 백엔드 테스트는 macOS 기본 `LibreSSL` 관련 `urllib3` 경고 1건만 남아 있고 실패는 없다.
